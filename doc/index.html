Dar documentation

DarSince release 2.0.0, dar is a command-line adaptation of the underlying libdar library. It uses libdar's API as well as several other external tools

Dar source code provides both dar and libdar, while many distro split each source code in two independent parts (dar and libdar), sometimes also seperating library headers and the current documentation in their own package.

the following documentation focuses on the command-line usage:

  • Tutorial
  • Mini-Howto
  • FAQ
  • command line usage notes
  • Man pages
  • Good Backup Practice short Guide
  • Sample and real Scripts

If after having read the previous documents, you still missing some information, you are welcome to ask for support

Libdar documentation

Libdar, the library that powers the dar command-line tool as well as several other softwares has its own documentation. All features available to dar are available thanks to libdar's API which is also documented with samples and tutorial.

The API exists today for C++ and Python3 languages.

  • The API tutorial
  • Dar command-line source code for API usage illustration
  • The libdar's python binding tutorial based on examples of use
  • The API reference documentation
  • The libdar-api mailing-list

Feedback and Interactions

Community
Dar has a community quite active on the dar-support mailing-list though it is not a substitute for trying finding your answers from the documentation first.
The dar-news mailing-list
is a read-only mailing-list that has a very low email volume (in average, less than one email a month). Its purpose is to informed about major events like new releases or security issues
Bug?
You thing you have hit a bug? If you are unsure, ask on the dar-support mailing-list else open a bur report at the Bug Tracker (if one has not already been openned for the same problem).
Contributing?
You want to contribute? You can either:
  • submit a path at the Patch Tracker
  • or propose a pull-request on github
  • Last you can also contribute by sharing your millions euro (I also accept dollars) with me :^)

In any case, thanks to respect the basic rule like coding style if possible document your code as described.
